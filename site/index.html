PLAY 01How it worksกลไกการทำงาน

Each plugin ships a UserPromptSubmit hook. When you hit enter, the hook injects its coaching rules into the turn — before Claude reads your request. That's why coaching can't be forgotten mid-session: the harness enforces it, not the model's memory. plugin แต่ละตัวมาพร้อม hook UserPromptSubmit — กด enter ปุ๊บ hook จะฉีดกติกาการโค้ชเข้าไปใน turn ก่อนที่ Claude จะอ่านคำขอของคุณ นี่คือเหตุผลที่การโค้ชไม่มีวันถูกลืมกลางคัน: harness เป็นคนบังคับ ไม่ใช่ความจำของ model

prompting-coach — opens the responseเปิดคำตอบ

  • Tune-up — one highest-leverage gap + a stronger prompt + the principle.Tune-up — จุดอ่อนที่คุ้มสุดหนึ่งจุด + prompt ที่แข็งแรงกว่า + หลักการเบื้องหลัง
  • Praise — names the practice you nailed, plus a level-up tip.Praise — บอกชื่อสิ่งที่คุณทำถูก พร้อมทิป level-up
  • Gate — severely under-specified substantial work stops for one question first.Gate — งานใหญ่ที่สเปคขาดหนัก หยุดถามหนึ่งคำถามก่อน

english-coach — closes the responseปิดท้ายคำตอบ

  • Thai prompt — corrected English + concise + idiomatic versions.Prompt ภาษาไทย — อังกฤษฉบับแก้แล้ว + แบบกระชับ + แบบ native
  • English with slips — marked up, with the why-tip in Thai.อังกฤษมีพลาด — แก้ให้พร้อมเหตุผลภาษาไทย

Both plugins are fail-open: if a hook ever breaks, your prompt passes through untouched and the skills stay manually invocable. They never touch your files — only the conversation. ทั้งสอง plugin เป็น fail-open: ถ้า hook พังเมื่อไหร่ prompt ของคุณผ่านไปตามปกติ และ skill ยังเรียกเองได้ ไม่แตะไฟล์ของคุณ — อยู่แค่ในบทสนทนาเท่านั้น

PLAY 03Limitationsข้อจำกัด

  • Hooks are POSIX sh scripts — Windows needs Git Bash (untested).hook เป็น POSIX sh script — Windows ต้องใช้ Git Bash (ยังไม่ได้ทดสอบ)
  • Fail-open: if a hook breaks, prompts pass through untouched and skills stay manually invocable.fail-open: ถ้า hook พัง prompt ผ่านตามปกติ และ skill ยังเรียกเองได้
  • Coaching adds a small number of output tokens to each turn.การโค้ชเพิ่ม output tokens เล็กน้อยในแต่ละ turn
  • Enforcement relies on the model following the hook's instruction to invoke the skill — smaller models (e.g. Haiku) may improvise the block's format instead of following the exact template. Sonnet and up follow reliably.การบังคับใช้อาศัย model ทำตามคำสั่งของ hook ให้เรียก skill — model เล็ก (เช่น Haiku) อาจแต่ง format เองแทนที่จะตาม template เป๊ะ ส่วน Sonnet ขึ้นไปทำตามได้เสถียรกว่า

PLAY 01How it works — the away fixtureกลไกการทำงาน — สนามเยือน

The Discord track runs on a self-hosted OpenClaw workspace — a config dir and a workspace dir, however you run the OpenClaw process (the Docker deployment example also persists a home volume for the Claude CLI login). This repo's openclaw/ folder is what gets deployed into that workspace: three Discord-flavored skills (coach, prompting-coach-discord, english-coach-discord) plus an AGENTS.md section. Unlike the CLI plugins' per-turn hook, OpenClaw has no per-message injection point — so AGENTS.md is loaded once at session start as stable context, and the coaching rules live there for the whole session instead. Delivery defaults to spoiler tags — the demos above are exactly what lands in the channel. Step-by-step commands to stand this up live in the kick-off section above. track Discord รันบน OpenClaw workspace แบบ self-host — มี config dir กับ workspace dir ไม่ว่าจะรัน OpenClaw process แบบไหนก็ได้ (ถ้าใช้ตัวอย่าง Docker deployment จะมี home volume แบบถาวรไว้เก็บ login ของ Claude CLI เพิ่มด้วย) โฟลเดอร์ openclaw/ ใน repo นี้คือสิ่งที่ deploy เข้า workspace นั้น: skill สาม flavour Discord (coach, prompting-coach-discord, english-coach-discord) บวก section ใน AGENTS.md ต่างจาก hook ฝั่ง CLI plugin ที่ฉีดกติกาทุก turn, OpenClaw ไม่มีจุด inject รายข้อความ — AGENTS.md เลยถูกโหลดครั้งเดียวตอน session เริ่มเป็น stable context กติกาการโค้ชเลยอยู่ตรงนั้นตลอดทั้ง session ค่าเริ่มต้นส่งแบบ spoiler tags — ตัวอย่างข้างบนคือสิ่งที่ลงใน channel จริงๆ คำสั่งแบบทีละขั้นสำหรับตั้งค่าอยู่ที่ ส่วน kick-off ด้านบน
